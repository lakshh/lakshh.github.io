A recent incident reinforced how I should approach working with AI tools.

Incident Overview

I was working with a colleague to solve a file-reading issue, so I turned to an AI tool for help. I shared the code, the logs, and the expected behavior. The AI tool recommended a fix that resolved the issue on the first run.

As we reviewed the code together, we considered how it would behave in different scenarios. During that analysis, we realized it could cause an out-of-memory error in an edge case.

First Lesson

Always review the code generated by an AI tool, no matter how correct it appears or how much time the review takes. The AI tool does not account for every possible flow and usage pattern in a production environment.

Tuning the AI

I asked the AI tool about the potential memory problem, and it acknowledged the issue. I then added instructions to the AI tool to always surface potential faults and trade-offs in its proposed code.

Second Lesson

You can improve AI output by adding relevant instructions that reflect your production scenarios and usage patterns.
